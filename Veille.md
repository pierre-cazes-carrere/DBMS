##### **D : Donnez une définition et des exemples de systèmes de gestion de**

##### **bases de données avec des illustrations.**



D\* : Une base de données est une collection organisée d’informations stockées électroniquement, permettant de stocker, retrouver, manipuler et gérer efficacement ces données. Elle contient des données structurées, semi-structurées ou brutes et souvent organisées sous forme de table avec des lignes et des colonnes.



Bases relationnelles : MySQL, PostgreSQL, Microsoft SQL Server. Ces bases utilisent des tables liées par des clés primaires et permettent des requêtes complexes via le langage SQL.



Bases NoSQL : MongoDB (documents JSON), Cassandra (magasin de colonnes larges). Elles gèrent mieux les données non structurées et sont adaptées aux gros volumes et à la scalabilité.



Bases multimodèles : ArangoDB, qui combinent plusieurs types de modèles (graphes, documents, relationnel) dans une seule plateforme.

##### **I : Définissez le langage SQL. Donnez les commandes les plus utilisées de**

I\* : SQL (Structured Query Language) est un langage informatique standardisé permettant de gérer et manipuler les bases de données relationnelles.



###### **COMMANDES ET JOINTURES**



Commandes SQL les plus utilisées :

SELECT : pour interroger les données d’une ou plusieurs tables.



INSERT : pour insérer de nouvelles lignes dans une table.



UPDATE : pour modifier des données existantes.



DELETE : pour supprimer des données.



CREATE TABLE : pour créer une nouvelle table.



ALTER TABLE : pour modifier la structure d’une table.



DROP TABLE : pour supprimer une table.



WHERE : pour filtrer les résultats selon une condition.



JOIN : pour combiner des lignes issues de plusieurs tables.



Types de jointures (JOIN) en SQL :

INNER JOIN : retourne les lignes qui ont des correspondances dans les deux tables.



LEFT JOIN (ou LEFT OUTER JOIN) : retourne toutes les lignes de la table de gauche et les correspondances de la table de droite, ou NULL si pas de correspondance.



RIGHT JOIN (ou RIGHT OUTER JOIN) : retourne toutes les lignes de la table de droite et les correspondances de la table de gauche, ou NULL si pas de correspondance.



FULL JOIN (ou FULL OUTER JOIN) : retourne toutes les lignes quand il y a une correspondance dans l’une ou l’autre des tables.



CROSS JOIN : produit le produit cartésien des deux tables, combinant chaque ligne de la première avec toutes les lignes de la seconde.
